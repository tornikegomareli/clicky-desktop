# Release 0.1 Checklist

## Scope

- Linux-first release
- Windows must still build in CI
- first-run setup flow must be usable
- local developer build must be documented

## Required Before Tagging

- [ ] `cargo check` passes
- [ ] `cargo test` passes
- [ ] PR CI passes on Linux and Windows
- [ ] README matches the actual product scope
- [ ] `.env.example` matches the active providers
- [ ] known limitations are documented
- [ ] packaging targets are chosen

## Known Non-Goals For 0.1

- signed installers
- polished Wayland-specific overlay backend
- fully hidden Linux overlay dock/taskbar presence in every compositor

## Follow-Up After 0.1

- package Linux artifacts
- package Windows artifacts
- split `src/overlay/renderer.rs`
- revisit compositor-specific overlay behavior
